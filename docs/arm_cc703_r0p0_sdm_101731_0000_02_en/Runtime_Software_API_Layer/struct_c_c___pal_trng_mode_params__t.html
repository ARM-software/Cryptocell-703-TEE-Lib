◆ adaptiveProportionCutOff

uint32_t CC_PalTrngModeParams_t::adaptiveProportionCutOff

The adaptive proportion cutoff, as defined in NIST SP 90B: Recommendation for the Entropy Sources Used for Random Bit Generation, section 4.4.2. This is calculated as C =CRITBINOM(W, power(2,(-H)),1-a), W = 1024, a = 2^(-40), H=(entropy per bit).

◆ numOfBytes

uint32_t CC_PalTrngModeParams_t::numOfBytes

The amount of bytes for the required entropy bits. It is calculated as ROUND_UP(ROUND_UP(((required entropy bits)/(entropy per bit)), 1024), (EHR width in Bytes)) / 8. The 1024 bits is the multiple of the window size. The multiple of the EHR width, which is 192 bits.

◆ repetitionCounterCutoff

uint32_t CC_PalTrngModeParams_t::repetitionCounterCutoff

The repetition counter cutoff, as defined in NIST SP 90B: Recommendation for the Entropy Sources Used for Random Bit Generation, section 4.4.1. This is calculated as C = ROUND_UP(1+(-log(W)/H)), W = 2^(-40), H=(entropy per bit).
